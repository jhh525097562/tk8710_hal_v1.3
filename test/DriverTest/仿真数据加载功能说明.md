# TK8710 仿真数据加载功能实现

## 功能概述
为 TestDriverSlave.c 添加了从指定路径读取仿真数据并通过 SPI 传输给 8710 芯片的功能。

## 实现的功能

### 1. 新增函数
- `load_and_send_simulation_data()`: 主要功能函数，负责读取文件并通过 SPI 传输数据

### 2. 支持的数据文件
从路径 `D:\data\K2301\K2301PHY_CaseList_Data\Simulation_Database\class3\case2\` 读取以下文件：

- **ANoise.txt**: 噪声数据 (8个 uint16_t 值)
  - 使用 infoType = 0 传输
  - 数据格式: 16位整数，转换为字节数组传输

- **GWRXAH.txt**: GWRXAH 数据 (128个用户，每个用户16个20bit AH值)
  - 使用 infoType = 1 传输  
  - 数据格式: 20bit整数，使用位操作紧凑打包为字节数组
  - 每个用户: 16 × 20bit = 320bit = 40字节
  - 128个用户: 128 × 40字节 = 5120字节
  - 按用户顺序打包，确保每个用户的16个AH数据保持连续

- **GWRxPilotPower.txt**: 导频功率数据 (128个用户，每个用户40bit PilotPower值)
  - 使用 infoType = 2 传输
  - 数据格式: 40bit整数，直接拆分为5字节传输
  - 每个用户: 40bit = 5字节
  - 128个用户: 128 × 5字节 = 640字节
  - 按用户顺序传输，大端序格式

- **TxFreq.txt**: 发射频率数据 (128个用户，每个用户32bit TxFreq值)
  - 使用 infoType = 3 传输
  - 数据格式: 32bit整数，直接拆分为4字节传输
  - 每个用户: 32bit = 4字节
  - 128个用户: 128 × 4字节 = 512字节
  - 按用户顺序传输，大端序格式

- **TxPower.txt**: 发射功率数据 (128个用户，每个用户8bit TxPower值)
  - 使用 infoType = 4 传输
  - 数据格式: 8bit整数，直接按字节传输
  - 每个用户: 8bit = 1字节
  - 128个用户: 128 × 1字节 = 128字节
  - 按用户顺序直接传输

### 3. 调用方式
- **自动调用**: 程序启动完成 HAL 初始化后自动调用一次
- **手动调用**: 在交互界面输入 'l' 或 'L' 命令可手动触发数据加载和传输

### 4. 错误处理
- 文件打开失败时显示错误信息并返回
- SPI 传输失败时显示错误信息并返回
- 数据解析失败时跳过该文件继续处理其他文件

## 技术实现细节

### 数据解析
- 使用 `fgets()` 逐行读取文件
- 使用 `sscanf()` 解析数值数据
- 支持空格、制表符分隔的多数值格式

### 数据转换
- 16位数据: 高字节在前，低字节在后
- 32位数据: 大端序格式 (高字节在前)

### SPI 传输
- 调用 `TK8710SpiSetInfo()` 函数进行数据传输
- 每种数据类型使用不同的 infoType 参数 (0-4)

## 使用说明

### 编译
使用原有的编译命令，新增功能已集成到 TestDriverSlave.c 中：

```bash
arm-buildroot-linux-gnueabihf-gcc -I../inc -I../port TestDriverSlave.c \
    ../port/tk8710_rk3506.c ../src/tk8710_irq.c ../src/tk8710_core.c \
    ../src/tk8710_config.c ../src/tk8710_log.c \
    -o TestDriverSlave -lgpiod -lpthread
```

### 运行
1. 确保数据文件存在于指定路径
2. 运行程序后会自动加载并传输数据
3. 可通过 'l' 命令手动重新加载数据

## 注意事项
1. 确保数据文件路径正确且文件存在
2. 确保已正确初始化 TK8710 芯片和 SPI 接口
3. 数据量较大时可能需要调整缓冲区大小
4. 传输失败会显示具体错误信息但不影响程序继续运行
5. GWRXAH 数据使用特殊的20bit打包算法，确保按用户顺序传输
6. GWRxPilotPower 数据使用40bit格式，每个用户5字节，大端序传输
7. TxFreq 数据使用32bit格式，每个用户4字节，大端序传输
8. TxPower 数据使用8bit格式，每个用户1字节，直接传输
9. 每次传输前都会打印前16字节的十六进制数据用于调试
10. 缓冲区大小已调整以容纳更多数据：GWRXAH(5120字节)、GWRxPilotPower(640字节)、TxFreq(512字节)、TxPower(128字节)
