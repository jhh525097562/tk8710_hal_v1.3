# 文档目的

本文档主要完成以下目标：

1. **MAC层功能定义**：明确定义TurMass™ Link系统中MAC层的完整功能体系，包括RLC无线链路控制、LLC逻辑链路控制、网络接入管理等功能模块的职责和接口。
2. **协议栈设计**：设计完整的MAC层协议栈架构，定义各层之间的交互关系、数据流转机制和协议处理流程，确保协议栈的层次清晰、职责明确。
3. **协议帧设计**：定义完整的MAC协议帧结构，包括帧头、帧体、帧尾等字段设计，支持LAN/WAN不同网络模式，满足各种业务场景的通信需求。

通过本文档，为TurMass™ MAC3.0协议的实现、测试和应用开发提供完整的技术规范和设计指导。

# 1 TurMass™ Link 系统简述

TurMass™ Link是一种高效、灵活的无线通信系统，专为满足多种物联网场景的需求而设计，支持多种组网形态和拓扑结构，提供可靠的通信能力和高效的资源管理。

**LAN网络：**

特点：局域覆盖与高频次交互

- 可控的局部空间内，追求通信的实时性和可靠性
- 高效的时隙调度（TDMA），减少碰撞和重传

**WAN网络：**

特点：广域覆盖与高容量

- 用低成本，覆盖广域空间
- 增加空间维度，提高系统容量
- 利用多天线技术，提高链路预算，降低干扰

## 1.2适用场景

- 无线抄表
- 农业
- 无线语音
- 高并发应用

## 1.3 与现有产品的关系

基于TurMass技术的芯片、模组和网关，面向多模态的组网应用和场景，定义一套完整的协议规范，TurMass MAC协议，构建统一的TurMass Link通信系统，适用于各种LPWAN应用场景。

# 2 设计需求与原则

## 2.1 背景

随着物联网（IoT）技术的快速发展，应用场景从传统的低功耗传感器网络扩展到高并发、低延迟的工业物联网和智慧城市等复杂场景。当前主流的无线通信协议（如LoRa、Zigbee等）在特定场景下表现优异，但在应对多样化组网需求和复杂部署环境时仍存在局限性。因此开发一套具备灵活组网能力、高效资源管理和强大适应性的MAC协议栈，成为满足未来物联网需求的关键。

## 2.2 设计目标

TML MAC 旨在构建一个兼容所有 TurMass™ 系统组网形态的高效通信架构，兼容低功耗、高可靠性和可扩展性等特性。通过时隙调度、动态路由、抗干扰机制和数据加密等操作，确保网络在复杂环境中的稳定性和安全性，同时支持大规模节点部署和灵活扩展。

## 2.3 系统定位

MAC层在TML系统中处于关键位置，承担着连接物理层与上层（如应用层）的桥梁作用。具体而言，MAC层在TML系统中的定位包括以下几个方面：

**资源管理与分配**：负责对无线信道资源进行高效管理和动态分配，确保大量终端设备能够公平、高效地共享有限的无线资源。

**接入控制与调度**：实现终端设备的接入控制机制，协调终端设备的接入请求，避免或减少接入碰撞，提升网络整体接入效率。

**数据传输控制**：负责数据包的封装、拆封、传输控制和差错控制，确保数据传输的可靠性和有效性。

**功耗管理**：通过优化终端设备的休眠和唤醒机制，降低终端设备的功耗，延长终端设备的电池寿命。

**Qos保障**：提供服务质量（Qos）保障机制，满足不同业务场景对延迟、可靠性、吞吐量等方面的差异化需求。

**安全性支持**：提供必要的安全机制，保障数据传输的安全性和完整性，防止非法接入和数据篡改。

综上所述，MAC层在TurMass™ Link系统中起着承上启下的关键作用，直接影响系统的整体性能和用户体验，是实现TurMass™ Link系统高效、可靠、低功耗运行的重要保障。

## 2.4 设计原则

**高效资源利用：**优化调度和资源分配算法，最大化频谱和能量资源利用率；

**模块化：**协议栈各功能组件可独立开发和测试，提高可维护性；

**低功耗优化：**基于休眠唤醒策略，最大限度地延长终端设备电池寿命。

**灵活性和可扩展性：**协议设计应具备良好的适应性，以支持不同的网络拓扑、应用场景和未来技术演进。

# 4 RLC功能定义

## 4.1 功能概述

RLC的核心职责是进行无线通信资源的调度策略，实现高效的介质访问控制。主要功能包括：

| **序号** | **功能名称** | **功能描述**                                      |
| -------------- | ------------------ | ------------------------------------------------------- |
| 1              | 信道接入           | 支持基于LBT的CSMA/CA冲突避免机制，支持同步/异步接入策略 |
| 2              | 资源调度           | 时域、频域、空域资源的统一调度策略                      |
| 3              | 链路适配           | 功率控制、速率控制等链路参数自适应策略                  |
| 4              | 低功耗设计         | 支持低功耗模式，支持定时唤醒、I/O唤醒和无线唤醒         |

## 4.2 信道接入策略

信道接入策略定义了设备如何获得无线信道使用权的方法和规则，是RLC层的核心功能之一。其目标是在保证公平接入的前提下，最大化信道利用率，支持不同业务场景的接入需求，同时避免冲突和干扰。

## 4.4 链路适配策略

链路适配策略定义了根据信道条件动态调整链路参数的方法和机制，是保障通信质量的重要手段。其目标是适应无线信道的变化特性，在保证可靠通信的前提下，优化传输效率和系统性能。

### 4.4.2 速率控制策略

**手动速率控制**：

支持通过网关命令或配置接口手动调整通信速率，适用于固定速率配置场景。

**自动速率控制(ADR)**：

**核心目标**：让每个终端使用刚好够用的最高速率，既保证可靠通信，又最大化网络容量。

**决策机制**：基于链路质量统计（RSSI、SNR、丢包率）计算链路余量，根据阈值决策升速、降速或保持当前速率。

**保护机制**：

- 边缘终端保护：最低速率保底，确保可达性
- 网络容量均衡：速率分组调度，避免低速率终端挤占资源
- 速率调整稳定性：迟滞机制、连续触发要求、调整冷却期

## 4.5 低功耗设计策略

低功耗设计策略定义了终端设备在不同工作模式下的功耗管理方法，是延长电池寿命的关键技术。其目标是在保证通信可靠性的前提下，最大限度降低设备功耗，满足物联网设备的长期运行需求。

**I/O唤醒**：外部事件触发唤醒，实现事件驱动模式。

**定时唤醒**：固定工作周期，休眠时关闭射频模块。

**无线唤醒**：网关发送特定唤醒信号，唤醒远端休眠节点。

**功耗优化策略**：

- **深度休眠**：在空闲时段进入最低功耗状态
- **快速唤醒**：优化唤醒时序，减少唤醒延迟
- **智能调度**：根据业务模式优化工作周期

## 5.2 工作机制

### 5.2.1 星型组网

所有终端节点直接与网关通信，适用于小规模、低功耗物联网场景。

**部署模式**：

- **网关+网络服务器模式**：网关负责数据转发，MAC功能由NS实现
- **网关独立模式**：网关独立运行MAC功能，处理后上传到AS

**主要流程**：

- **入网流程**：设备白名单注册→节点启动→入网请求→入网应答→入网完成
- **数据传输**：支持上行/下行数据传输和确认重传机制
- **广播机制**：网关向所有终端发送网络配置、时间同步等信息
- **网络管理**：支持主动/被动离网，释放网络资源

### 5.2.2 中继拓扑

中继设备扩展网络覆盖范围，支持远距离通信。

**主要流程**：

- **入网流程**：中继先入网→终端搜索网关/中继→终端通过中继入网
- **数据转发**：终端↔中继↔网关的双向数据转发，支持确认重传
- **层级管理**：预配置设备层级，避免冲突和环路

## 5.3 帧封装与解析

支持MAC帧的封装与解析，包括帧头、帧体、帧尾的组装和拆解，确保数据格式一致性。

**对应帧字段**：

- **MHDR**（[6.1.1](#611-mhdr)）：MAC帧头，包含帧类型、设备类型、协议版本等信息
- **FHDR**（[6.1.2.1](#6121-fhdr)）：帧头，包含地址、控制、序号等字段
- **MIC**（[6.1](#61-帧格式)）：消息完整性校验码，用于数据完整性验证

## 5.4 分包与组包策略

根据时隙长度和数据包大小，支持动态分包策略：

- **分包策略**：大包分小包，适应时隙限制
  - **对应帧字段**：**SubPkg Field**（[6.1.2.1](#6121-fhdr)）：指示分包域存在，支持大包分小包
- **组包策略**：小包组装成完整数据包
- **序号管理**：确保包序连续性和完整性
  - **对应帧字段**：**FCnt**（[6.1.2.1](#6121-fhdr)）：帧序号，确保包序连续性

## 5.5 确认与重传机制

**确认机制**：

- **ACK确认**：接收方正确接收后发送确认
  - **对应帧字段**：**FCtrl.ACK**（[6.1.2.1](#6121-fhdr)）：确认标识，支持确认帧
- **NACK确认**：接收方检测到错误时发送否定确认

**重传策略**：

- **超时重传**：发送方未在规定时间内收到确认时重传
- **选择性重传**：仅重传丢失或损坏的数据包
  - **对应帧字段**：**FCtrl.Retran_Pkg**（[6.1.2.1](#6121-fhdr)）：指示重传域存在
  - **对应帧字段**：**Retrans Field**（[6.1.2.1](#6121-fhdr)）：重传域，标识重传包
- **重传次数限制**：避免无限重传造成网络拥塞

## 5.6 广播与组播

**广播机制**：

- **全网广播**：网关向所有终端发送相同信息
  - **对应帧字段**：**FrameType=0101**（[6.1.1](#611-mhdr)）：广播帧类型标识
- **广播时隙**：专用时隙用于广播消息传输

**组播机制**：**（待定）**

## 5.7 安全机制

**数据加密**：

- **端到端加密**：数据传输过程中全程加密
  - **对应帧字段**：**SecurityMode**（[6.1.1](#611-mhdr)）：安全模式，支持TEA/AES128加密
- **密钥管理**：支持网络会话密钥、应用会话密钥管理

**完整性校验**：

- **MIC校验**：消息完整性校验码验证数据完整性
  - **对应帧字段**：**MIC**（[6.1](#61-帧格式)）：消息完整性校验码
- **防篡改**：检测数据传输过程中的篡改

**防重放攻击**：

- **序列号机制**：基于序列号检测重放攻击
  - **对应帧字段**：**FCnt**（[6.1.2.1](#6121-fhdr)）：帧序号，检测重放攻击
- **时间戳验证**：防止旧消息重放

## 5.8 Qos服务机制

**优先级管理**：

- **业务分级**：按业务重要性设置不同优先级
  - **对应帧字段**：**QosPri**（[6.1.1](#611-mhdr)）：Qos优先级，业务重要性分级
- **优先级队列**：高优先级业务优先处理

**资源保障**：

- **时隙优先**：高优先级业务优先获得时隙资源
  - **对应帧字段**：**QosTTL**（[6.1.1](#611-mhdr)）：Qos生存时间等级

## 5.9 接入与设备管理

**设备标识体系**：

- **DevEUI**：设备唯一标识（8字节）
- **DevAddr**：网络地址（2/4字节）
- **DevType**：设备类型标识（1字节）
  - **对应帧字段**：**DevType**（[6.1.1](#611-mhdr)）：设备类型：终端/网关/中继

**设备发现**：

- **被动发现**：通过入网请求发现新设备
- **主动发现**：网关广播探测，终端响应

**设备状态维护**：

- **在线监测**：心跳/保活机制监测设备状态
- **状态管理**：在线、离线、休眠、故障等状态
- **生命周期管理**：设备添加、更新、删除、老化清理

**资源管理**：

- **时隙绑定**：设备入网后分配专用时隙
- **资源回收**：超时自动释放时隙等网络资源

**地址管理**：

- **对应帧字段**：**Src Addr/Dst Addr**（[6.1.2.1](#6121-fhdr)）：源/目标设备地址
- **对应帧字段**：**AddrMode**（[6.1.1](#611-mhdr)）：地址模式：短地址/长地址

**网络模式管理**：

- **对应帧字段**：**NwkMode**（[6.1.1](#611-mhdr)）：网络模式：LAN/WAN/Satellite/Mesh

**中继拓扑支持**：

- **对应帧字段**：**HopNum、HopCnt**（[6.1.1](#611-mhdr)）：最大跳数/当前跳数，支持中继

**入网流程管理**：

- **对应帧字段**：**FrameType=0000/0001**（[6.1.1](#611-mhdr)）：Join Request/Accept帧类型

**离网管理**：

- **对应帧字段**：**FrameType=0010**（[6.1.1](#611-mhdr)）：Disassociation帧类型

MHDR部分字段说明如下：

**AddrMode– 地址类型字段：**

- 0：短地址模式，此模式下FHDR的Src/Dst Addr填充2字节NwkAddr；
- 1：长地址模式，此模式下FHDR的Src/Dst Addr填充4字节NwkID+NwkAddr；

默认情况源地址和目标地址填充2字节，短地址模式下选择NwkAddr作为地址。

**HopLimit – 跳数限制字段：**

- 当网络类型为星型网络时，HopLimit字段默认填充全0；
- 当网络类型为Mesh网络时，HopLimit字段定义为网络最大跳数限制，若路由节点检测到当前所经跳数达到最大跳数限制，对该数据包作丢弃处理。

**HopCnt – 数据跳数/设备层级字段：**

- 当网络类型为星型网络且设备与网关直连时，HopCount字段应为全0；
- 当网络类型为星型网络且设备通过中继与网关通信时（中继拓扑），HopCount字段表示当前数据所经中继转发层级；
- 当网络类型为Mesh网络时，HopCount字段表示数据当前所经路由转发跳数

**NwkMode - 网络模式**

- 当网络模式为WAN时

  - MHDR中HopNum和HopCnt均为2bit，新增2bit的QosPri和2bit的QosTTL。
  - MACPayload中含PayLoadLen字段（**PHY对Payload不做32bit的crc**）
  - 入网accept中，使用**WRSBGroup**字段
  - 广播中使用**BroadcastGroup（WAN）**字段
- 当网络模式为LAN时

  - MHDR中HopNum和HopCnt均为4bit
- 当网络模式为Satellite Network时

  - 字段只有3byte的MHDR、FHDR中只有4byte的Src Addr、2byte的PayLoadLen

### 6.2.6 Broadcast（LAN）

Broadcast用于节点向网络中的所有节点发送广播消息，广播消息的Dst Addr目标地址应填充为全F，对应的MAC Payload定义见5.1.2小节的描述。

为保障不同业务场景节点入网、数据收发的可靠性，TML MAC应支持在广播中携带部分扩展信息（如网关当前时隙配置），从而实现在有干扰环境下网络节点也能够通过网关广播信息动态调整运行策略，其中广播的FPayload字段定义如下：

| **Bytes/Bit** | **字段名称** | **字段长度（bits）** | **说明**          |
| -------------------- | ------------------ | -------------------------- | ----------------------- |
| 1                    | BroadcastCtrl      | /                          | 广播控制字段            |
| 0~599                | BroadcastGroup     | /                          | 广播信息组，WAN：固定15 |

其中广播FPayload部分字段定义如下：

**BroadcastCtrl – 广播控制字段：**

| **Bit** | **字段名称** | **字段长度（bits）** | **说明**                     |
| -------------- | ------------------ | -------------------------- | ---------------------------------- |
| 7:4            | RFU                | 4                          | 保留                               |
| 3:0            | BGroupNum          | 4                          | 广播携带信息类型数量，WAN：固定为1 |

**BroadcastGroup – 广播信息组（LAN）：**

- 广播信息组仅当BGroupNum字段非零时存在，标识当前帧携带的广播信息
- 广播信息组定义遵循TLV（Type-Length-Value）格式，根据BGroupNum字段决定携带多少种扩展信息，具体定义如下：

| **Bytes/Bit** | **字段名称** | **字段长度（bits）** | **说明**    |
| -------------------- | ------------------ | -------------------------- | ----------------- |
| 2                    | BOptsCtrl_1        | /                          | 广播信息1控制字段 |
| 1~599                | BOptsPayload_1     | /                          | 广播信息1负载信息 |
| …                   |                    |                            |                   |
| 2                    | BOptsCtrl_N        | /                          | 广播信息N控制字段 |
| 1~599                | BOptsPayload_N     | /                          | 广播信息N负载信息 |

- 其中BGroupCtrl字段的定义如下：

| **Bit** | **字段名称** | **字段长度（bits）** | **说明**                                                                                                                     |
| -------------- | ------------------ | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| 15:9           | BOptsType          | 7                          | 扩展信息类型：  0000000：设置时隙结构  0000001：主通道频点列表  0000010：从通道频点列表  0000011：上行占位bit表  0000100：唤醒配置 |
| 8:0            | BOptsLen           | 9                          | 扩展信息长度                                                                                                                       |

- 其中对于不同BOptsType，具体的BOptsPayload定义如下：

**0000000 – 网关时隙结构：**

- 网关时隙结构按照每个时隙2 bytes进行传输，根据BOptsLength字段指示，即可自动计算时隙数量，具体一个时隙结构字段定义如下：

| **Bit** | **字段名称** | **字段长度（bits**） | **说明**                                                                                                         |
| -------------- | ------------------ | -------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| 15:13          | SlotType           | 3                          | 时隙类型：  000：TX_BCN  001：RX_BCN  010：TX_DATA  011：RX_DATA  100：MS_IDLE  101：BCN_IDLE                          |
| 12:10          | SlotRate           | 3                          | 时隙速率：  000：RATE_6  001：RATE_7  010：RATE_8  011：RATE_9  100：RATE_10  101：RATE_11  110：RATE_18  111：RATE_24 |
| 9:0            | SlotLen            | 10                         | 时隙长度                                                                                                               |

- 其中BCN_IDLE和DATA_IDLE为配置为字节长度的IDLE，根据速率模式和字节长度计算IDLE时隙长度，MS_IDLE为配置为时间长度的IDLE（单位ms）

**0000001 –**主通道频点列表**：**

- 本网关从通道频点是当前主通道携带的从通道中心频点，终端节点将优先遍历从通道频点进行入网流程，按照每个频点4bytes进行传输，可自动计算从通道频点数量，频点字段定义如下：

| **Bytes/Bit** | **字段名称** | **字段长度（bits）** | **说明** |
| -------------------- | ------------------ | -------------------------- | -------------- |
| 4                    | SChanFreq          | /                          | 从通道频点     |

**0000010 –**从通道频点列表**：**

- 其他网关从通道频点是其他从网关携带的自身所有主通道携带的部分从通道频点（如首个从通道频点），终端在遍历本网关全部从通道频点后，若无有效时隙可占用，则继续遍历其他网关从通道频点，频点定义字段同本网关从信道频点。

**0000011 – 占位符信息：**

- 占位符信息旨在描述当前本网关的上行时隙占位信息，采用bitmap形式，对每一个上行时隙保留1bit的占位信息，若终端节点解析到当前上行时隙已被占用，则自动切换到下一个从通道频点进行入网流程。
- 占位符的长度（字节数）由当前网关全时隙数量决定，如网关当前有8个上行时隙，则占位符长度为1字节；若网关当前有10个时隙，则占位符长度为2字节。

# 7 工作流程

本章描述TurMass™ MAC协议的主要工作流程，包括搜网、入网、数据传输、离网、广播、重传和低功耗唤醒等流程。



## 7.1 搜网流程

终端上电后搜索网关广播信号，获取网络配置信息。

```mermaid
sequenceDiagram
    participant TE as 终端
    participant GW as 网关
    
    TE->>TE: 上电初始化，配置默认速率和信道组
    
    loop 遍历信道组内所有信道
        TE->>TE: 切换到下一个信道
        GW->>TE: 周期发送BCN/广播（含TddFrameNum、FreqGroupNum等）
        
        alt 接收到广播
            TE->>TE: 解析广播，获取时隙配置信息
            TE->>TE: 搜网成功，启动入网流程
        else 超时未收到
            TE->>TE: 继续遍历下一信道
        end
    end
    
    alt 所有信道遍历完成仍失败
        TE->>TE: 切换速率，重新搜网
    end
```



## 7.2 入网流程

终端向网关/NS发起入网请求，获取网络地址和资源分配。

```mermaid
sequenceDiagram
    participant TE as 终端
    participant GW as 网关
    participant NS as NS
    
    TE->>TE: 搜网成功，准备入网
    TE->>GW: Join Request（DevEUI、DevNonce、Capacity）
    GW->>NS: 转发Join Request
    
    NS->>NS: 验证设备合法性（白名单检查）
    NS->>NS: 分配NwkAddr、生成JoinNonce
    NS->>NS: 分配资源（TddFrameIdx、FreqGroupIdx等）
    
    NS->>GW: Join Accept（NwkAddr、JoinNonce、WRBGroup）
    GW->>TE: 转发Join Accept
    
    alt 入网成功（Status=0）
        TE->>TE: 保存NwkAddr、计算会话密钥
        TE->>TE: 配置分配的时频资源
        TE->>TE: 入网完成，进入工作状态
    else 入网失败（Status=1）
        TE->>TE: 等待退避时间后重试
    end
```



## 7.3 离网流程

终端主动离网或被NS强制离网，释放网络资源。

```mermaid
sequenceDiagram
    participant TE as 终端
    participant GW as 网关
    participant NS as NS
    
    alt 主动离网（LeaveMode=00）
        TE->>TE: 触发离网（用户操作/应用请求）
        TE->>GW: Disassociation（DevEUI、LeaveMode=00）
        GW->>NS: 转发离网请求
        NS->>NS: 释放NwkAddr、回收时频资源
        TE->>TE: 清除会话密钥，进入离线状态
        
    else 被动离网（LeaveMode=01）
        NS->>NS: 检测到设备异常/管理员操作
        NS->>GW: Disassociation（DevEUI、LeaveMode=01）
        GW->>TE: 转发离网命令
        TE->>TE: 清除会话密钥，进入离线状态
        NS->>NS: 释放NwkAddr、回收时频资源
    end
```



## 7.4 单包数据上行流程

sequenceDiagram

    participant GW as 网关
    participant NS as NS

    end

## 7.6 广播发送流程

网关周期性发送广播消息，用于网络同步和配置下发。支持两种模式：网关自主管理广播和NS下发Payload广播。

```mermaid
sequenceDiagram
    participant NS as NS
    participant GW as 网关
    participant TE as 终端
    
    NS->>GW: 配置广播参数（TddFrameNum、RateNum等）
    
    alt 无Payload广播（网关自主管理）
        loop 每个超帧周期
            loop 每个时帧（TddFrameIdx = 0 ~ TddFrameNum-1）
                GW->>GW: 构建广播帧，更新TddFrameIdx
                
                alt WAN多天线模式
                    GW->>GW: 天线轮发广播（空间分集增益）
                end
                
                GW->>TE: Broadcast（FrameType=0101，Dst Addr=0xFFFF）
                TE->>TE: 解析广播，同步时帧序号
                GW->>GW: 时帧结束，TddFrameIdx++
            end
            GW->>GW: 超帧结束，TddFrameIdx重置为0
        end
        
    else 带Payload广播（NS下发）
        NS->>GW: 下发广播帧（含BroadcastPayload）
        GW->>GW: 在当前时帧序列中插入Payload
        
        alt WAN多天线模式
            GW->>GW: 天线轮发广播（空间分集增益）
        end
        
        GW->>TE: Broadcast（含BroadcastPayload）
        TE->>TE: 解析广播，处理Payload内容
        Note over GW: Payload仅发送一次，后续恢复自主管理
    end
```



## 7.7 分包/重传流程

**WAN特殊说明**：WAN对所有分包只做一次ACK，只在最后一个分包传输完成后进行确认，而不对各个分包单独回复ACK



### 7.7.1 分包流程

```mermaid
sequenceDiagram
    participant APP as 应用层
    participant TE as 终端MAC
    participant GW as 网关
    participant NS as NS
    APP->>TE: 发送大数据包（长度 > 单包最大长度）
    Note over TE: 分包处理
    TE->>TE: 计算总分包数SubPkgNum
    TE->>TE: 设置FCtrl.Sub_Pkg=1
    loop 发送每个分包（SubPkgIdx从1到SubPkgNum）
        TE->>TE: 设置SubPkgIdx=当前分包序号、SubPkgNum=总分包数、FCnt=N+SubPkgIdx-1
        TE->>GW: Confirm Data（SubPkg Field）
        GW->>NS: 转发分包数据
        NS->>NS: 缓存分包，等待组包
    end
    Note over NS: 组包处理
    NS->>NS: 检查所有分包是否到齐
    alt 所有分包接收完成
        NS->>NS: 按SubPkgIdx顺序组装完整数据
        NS->>NS: MIC校验、解密、处理数据
        NS->>GW: ACK响应（确认所有分包）
        GW->>TE: 转发ACK（FCtrl.ACK=1）
        TE->>TE: 分包传输完成
        TE->>APP: 上报发送成功
    else 分包丢失或超时
        NS->>NS: 记录丢失的分包序号
        NS->>GW: NACK响应（指示丢失分包）
        GW->>TE: 转发NACK
        TE->>TE: 触发选择性重传
    end
```



### 7.7.2 重传流程

```mermaid
sequenceDiagram
    participant TE as 终端MAC
    participant GW as 网关
    participant NS as NS
    
    TE->>GW: Confirm Data（FCnt=N）
    GW->>NS: 转发上行数据
    NS->>NS: MIC校验、解密、处理数据
    
    alt 传输成功
        NS->>GW: ACK响应
        GW->>TE: 转发ACK（FCtrl.ACK=1）
        TE->>TE: 传输完成
        
    else 超时未收到ACK
        Note over TE: 启动重传机制
        TE->>TE: 设置FCtrl.Retran_Pkg=1、初始化RetransIdx=1
        TE->>TE: 设置RetransNum=最大重传次数
        
        loop RetransIdx ≤ RetransNum
            TE->>TE: 设置Retrans Field
            TE->>GW: 重传数据（含Retrans Field）
            GW->>NS: 转发重传数据
            NS->>NS: 根据FCnt去重，处理数据
            
            alt 收到ACK
                NS->>GW: ACK响应
                GW->>TE: 转发ACK（FCtrl.ACK=1）
                TE->>TE: 重传成功，传输完成
            else 仍超时
                TE->>TE: RetransIdx++
                TE->>TE: 等待退避时间后继续重传
            end
        end
        
        alt 达到最大重传次数仍失败
            TE->>TE: 重传失败
            TE->>TE: 上报应用层传输失败
        end
    end
```



### 7.7.3 分包重传流程（选择性重传）

```mermaid
sequenceDiagram
    participant TE as 终端MAC
    participant GW as 网关
    participant NS as NS
    
    Note over TE,NS: 分包传输过程中部分分包丢失
    
    TE->>GW: 分包1（SubPkgIdx=1, SubPkgNum=4）
    GW->>NS: 转发分包1
    TE->>GW: 分包2（SubPkgIdx=2, SubPkgNum=4）
    Note over GW: 分包2传输失败

    GW->>NS: 转发分包3
    TE->>GW: 分包4（SubPkgIdx=4, SubPkgNum=4）
    GW->>NS: 转发分包4
    
    NS->>NS: 检测到分包2丢失
    NS->>GW: NACK（指示缺少分包2）
    GW->>TE: 转发NACK

```

## 7.9 速率调整流程

终端MAC协议栈根据链路质量自适应调整通信速率，NS在入网响应中提供参考工作速率。

```mermaid
sequenceDiagram
    participant TE as 终端
    participant GW as 网关
    participant NS as NS
    
    Note over TE,NS: 入网阶段：获取参考速率
    TE->>GW: Join Request（使用默认最低速率）
    GW->>NS: 转发Join Request
    NS->>NS: 根据终端能力/信号强度分配资源
    NS->>GW: Join Accept（含WorkRateMode）
    GW->>TE: 转发Join Accept
    TE->>TE: 解析WorkRateMode，设置初始工作速率
    
    Note over TE,NS: 工作阶段：自适应速率调整
    loop 数据传输过程
        TE->>TE: 统计链路质量（RSSI、SNR、丢包率）
        TE->>TE: 计算链路余量
        
        alt 链路余量 > 升速阈值（连续N次）
            TE->>TE: 升速决策，切换到更高速率
            TE->>TE: 更新UpBlockNum、DownBlockNum
            Note over TE: 使用Rate0~Rate3对应的配置
            
        else 链路余量 < 降速阈值
            TE->>TE: 降速决策，切换到更低速率
            TE->>TE: 更新UpBlockNum、DownBlockNum
            Note over TE: 边缘终端保护：最低速率保底
            
        else 链路余量在正常范围
            TE->>TE: 保持当前速率
        end
        
        TE->>GW: 使用调整后的速率发送数据
        GW->>NS: 转发数据
        NS->>GW: 下行响应（速率与上行一致）
        GW->>TE: 转发下行数据
        
        TE->>TE: 进入调整冷却期，避免频繁切换
    end
```











## 7.10 闭环功控流程

基于接收端反馈进行功率调整，实现精确功率控制。

```mermaid
sequenceDiagram
    participant TE as 终端
    participant GW as 网关
    participant NS as NS
    
    Note over TE,NS: 前提：MHDR.PowerControlType = 1（闭环功控）
    
    TE->>TE: 设置当前发送功率TxPowerIdx
    TE->>GW: 上行数据（FHDR含TxPowerIdx=当前发送功率）
    GW->>NS: 转发上行数据和RSSI信息
    
    NS->>NS: 根据接收RSSI计算链路余量
    NS->>NS: 计算期望终端发送功率
    
    alt RSSI过高（功率冗余）
        NS->>NS: 计算降低功率值
        Note over NS: 期望功率 = 当前功率 - 调整步长
        
    else RSSI过低（功率不足）
        NS->>NS: 计算提升功率值
        Note over NS: 期望功率 = 当前功率 + 调整步长
        
    else RSSI在目标范围
        NS->>NS: 保持当前功率
    end
    
    NS->>GW: 下行数据（FHDR含TxPowerIdx=期望终端功率）
    GW->>TE: 转发下行数据
    
    TE->>TE: 解析下行TxPowerIdx
    TE->>TE: 计算期望功率：-40 + TxPowerIdx × 4 dBm
    TE->>TE: 调整发送功率到期望值
    
    TE->>GW: 下次上行使用调整后的功率发送
    Note over TE,NS: 循环迭代，逐步收敛到最优功率
```



## 7.11 QoS服务流程

基于业务优先级的服务质量保障机制，确保重要业务优先传输。

```mermaid
sequenceDiagram
    participant APP as 应用层
    participant TE as 终端MAC
    participant GW as 网关
    participant NS as NS
    
    Note over APP,NS: 上行QoS流程
    
    APP->>TE: 业务数据（含优先级标识）
    TE->>TE: 根据业务类型设置QosPri（0~3）
    TE->>TE: 设置QosTTL生存时间等级
    
    alt 多业务并发
        TE->>TE: 数据入队到对应优先级队列
        Note over TE: 终端维护多缓存队列
        TE->>TE: 按优先级排序，高优先级优先发送
    end
    
    TE->>GW: 上行数据（MHDR含QosPri、QosTTL）
    GW->>NS: 转发上行数据
    NS->>NS: 解析QosPri，按优先级处理
    
    Note over APP,NS: 下行QoS流程
    
    NS->>NS: 下行数据到达，设置QosPri
    NS->>GW: 下发数据（含QosPri、QosTTL）
    
    alt 多下行数据并发
        GW->>GW: 数据入队到对应优先级队列（4级队列）
        GW->>GW: 按队列优先级调度发送
        Note over GW: 高优先级队列优先获得时隙资源
    end
    
    GW->>TE: 下行数据（在分配时隙发送）
    TE->>TE: 接收处理
    
    Note over APP,NS: QosTTL超时处理
    
    alt QosTTL超时
        GW->>GW: 检测数据生存时间
        GW->>GW: 丢弃超时数据，释放队列资源
    end
```

## 7.12 漫游切换流程

### 7.12.1 被动漫游切换流程

当原网关故障或无响应时，终端被动切换到新网关。LAN和WAN网络均支持此流程。

```mermaid
sequenceDiagram
    participant TE as 终端
    participant GW1 as 原网关（故障）
    participant GW2 as 新网关
    participant NS as NS
    
    Note over TE,GW1: 原网关故障或无响应
    
    TE->>GW1: 上行数据
    Note over GW1: 网关故障，无响应
    TE->>TE: 等待ACK超时
    
    TE->>GW1: 重传数据（RetransIdx=1）
    Note over GW1: 仍无响应
    TE->>TE: 重传超时
    
    loop 重传次数达到上限
        TE->>GW1: 继续重传
        TE->>TE: 仍无响应
    end
    
    TE->>TE: 判定原网关失联、进入漫游扫描模式
    
    Note over TE: 扫描可用网关
    TE->>TE: 扫描广播信道、发现新网关GW2
    
    Note over TE,NS: 向新网关发起入网
    TE->>GW2: Join Request（DevEUI、DevNonce）
    GW2->>NS: 转发入网请求
    NS->>NS: 验证设备身份、分配新接入参数
    
    NS->>GW2: Join Accept（新DevAddr、SlotGroupIdx、PointFreqIdx）
    GW2->>TE: 转发Join Accept
    TE->>TE: 更新网络参数、完成被动漫游切换
    
    TE->>GW2: 上行数据（恢复通信）
    GW2->>NS: 转发数据
    NS->>NS: 漫游切换完成
```



### 7.12.2 无缝漫游流程（WAN同频同步组网）

WAN网络采用同频同步组网，多个网关使用相同的频率和时隙结构，终端无需重新入网即可实现无缝漫游。

**WAN同频同步组网特点**：

- 所有网关使用相同的工作频率和时隙结构
- 终端入网参数（SlotGroupIdx、PointFreqIdx）在整个网络内有效
- NS统一管理所有网关，终端数据可被任意覆盖范围内的网关接收
- 终端无感知切换，无需重新入网

**无缝漫游关键机制**：

- **同频配置**：所有网关配置相同的工作频率，终端无需切换频率
- **同步时隙**：所有网关使用统一的时隙结构，终端按原有时隙发送
- **全网有效参数**：入网时分配的SlotGroupIdx、PointFreqIdx在整个网络内有效
- **NS统一调度**：NS根据接收数据的网关自动更新终端关联关系
- **最优下行选择**：NS选择信号质量最优的网关进行下行数据发送

```mermaid
sequenceDiagram
    participant TE as 终端
    participant GW1 as 网关1
    participant GW2 as 网关2
    participant NS as NS
    
    Note over TE,NS: WAN同频同步组网配置
    
    NS->>GW1: 配置同频同步参数
    NS->>GW2: 配置同频同步参数（与GW1相同）
    Note over GW1,GW2: 所有网关使用相同频率和时隙结构
    
    Note over TE,NS: 终端入网（通过GW1）
    TE->>GW1: Join Request
    GW1->>NS: 转发入网请求
    NS->>NS: 分配全网有效的接入参数
    NS->>GW1: Join Accept（SlotGroupIdx、PointFreqIdx）
    GW1->>TE: 转发Join Accept
    TE->>TE: 保存入网参数
    
    Note over TE,NS: 正常通信（GW1覆盖区域）
    TE->>GW1: 上行数据
    GW1->>NS: 转发数据
    NS->>GW1: 下行数据
    GW1->>TE: 转发下行数据
    
    Note over TE: 终端移动到GW2覆盖区域
    
    Note over TE,NS: 无缝漫游（无需重新入网）
    TE->>GW2: 上行数据（使用原有入网参数）
    Note over GW2: 同频同步，可直接接收
    GW2->>NS: 转发数据（携带网关标识）
    NS->>NS: 更新终端当前关联网关为GW2
    NS->>GW2: 下行数据
    GW2->>TE: 转发下行数据
    
    Note over TE,NS: 终端无感知完成漫游切换
    
    alt 终端在GW1和GW2覆盖重叠区域
        TE->>GW1: 上行数据
        TE->>GW2: 上行数据（同时被两个网关接收）
        GW1->>NS: 转发数据
        GW2->>NS: 转发数据
        NS->>NS: 数据去重
        NS->>NS: 选择信号质量更优的网关作为下行通道
    end
```

## 8.2 LAN、WAN的字段差异

### 8.2.1 数据帧格式差异

数据帧结构（LAN）：

| MHDR   | MACPayload | MIC      |
| ------ | ---------- | -------- |
| 3Bytes | 7-597Bytes | 0/2Bytes |

数据帧结构（WAN）：

| MHDR   | PayLoadLen | MACPayload | MIC      |
| ------ | ---------- | ---------- | -------- |
| 3Bytes | 2Bytes     | 7-597Bytes | 0/2Bytes |

### 8.2.2 字段差异

| MHDR字段         | LAN                                                    | WAN                                                                                                       |
| ---------------- | ------------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
| HopCnt           | 4bit                                                   | 2bit                                                                                                      |
| HopNum           | 4bit                                                   | 2bit                                                                                                      |
| QosPri           | 无                                                     | 新增2bit QosPri：Qos优先级                                                                                |
| QosTTL           | 无                                                     | 新增2bit QosTTL:Qos生存时间等级                                                                           |
| PowerControlType | 无                                                     | 新增1bit功控类型：<br />0：开环功控<br />1：闭环功控                                                      |
| NwkMode          | 当NwkMode为LAN时，入网accept及广播中，使用LANGroup字段 | 当NwkMode为WAN时，入网accept和广播中，使用WANGroup资源组<br />当NwkMode为WAN时，新增2byte的PayloadLen字段 |

| FHDR字段   | LAN                           | WAN                                                                                                |
| ---------- | ----------------------------- | -------------------------------------------------------------------------------------------------- |
| TxPowerIdx | 无                            | MHDR中功控类型为闭环，则新增4bit闭环功控的字段，上行时添当前发送功率；下行时填期望终端上行发送功率 |
| ACK        | 有ACK时，LAN对每个分包都回ACK | 有ACK时，WAN对所有包一起回ACK，不在分包中回ACK                                                     |

### 8.2.4 入网accept帧类型差异

| **字段名称** | LAN                                                                          | WAN                        |
| ------------------ | ---------------------------------------------------------------------------- | -------------------------- |
| JoinResult         | 时隙资源块分配结果                                                           | 时隙、频域等资源块分配结果 |
| SRBGroup           | LAN：时隙资源块组：0-30bytes                                                 | WAN时隙资源块：7bytes      |
| GWID               | 网关唯一标识号，仅LAN有                                                      | 无                         |
| WakeUpCfg          | 低功耗设备携带，标识当前休眠唤醒配置信息，定义参考3.1.2.1章节扩展域，仅LAN有 | 无                         |

### 8.2.5 广播帧类型差异

| MHDR   | BroadcastCtrl | BroadcastGroup |

| 3Bytes | 1Byte         | 0~599Byte      |

广播帧结构（WAN）：

| MHDR   | BroadcastGroup | BrocastPayloadLen | BrocastPayload |
