## 1. TK8710简介

### 1.1 芯片概述

TK8710是一款纯数字的多天线收发器，支持8组射频通道、128个数据包（用户）的并发处理。外部主控可以通过SPI接口来控制和访问TK8710，实现数据收发。

该芯片集成了先进的数字信号处理单元、多天线信号处理、波束成形处理技术，能够在复杂的无线环境中提供稳定、高效的数据传输能力。TK8710采用SPI接口与主控芯片通信，支持中断方式下的异步调用，可与多种嵌入式平台无缝集成。

### 1.2 主要特性

| 特性类别           | 详细说明                                |
| ------------------ | --------------------------------------- |
| **芯片封装** | BGA324，尺寸15x15 mm                    |
| **通信频段** | 400MHz~510MHz / 862MHz~960MHz          |
| **天线配置** | 支持1、2、4、8天线配置                  |
| **用户容量** | 最大支持128个用户同时收发               |
| **灵敏度**   | -146 dBm（mode5 @0.9Kbps，PER=5%）      |
| **信号带宽** | 62.5 kHz ~ 500kHz                       |
| **接口类型** | 4线SPI slave接口，最大16Mbps            |
| **时隙结构** | 支持4时隙TDD帧结构（BCN + 3个数据时隙） |
| **调制方式** | 支持多种速率模式（Rate 5-11, 18）       |
| **工作模式** | 主模式（Master）/ 从模式（Slave）       |
| **工作时钟** | 32MHz                                   |
| **工作电压** | 1.8 ~ 3.6 V                             |
| **典型功耗** | ≤ 100 mA                               |

## 2. TK8710 HAL概述

TK8710 HAL（Hardware Abstraction Layer，硬件抽象层）是专为TK8710多天线收发器芯片设计的抽象层。该HAL层位于MAC层和硬件层之间，为上层应用提供统一、简洁的编程接口，有效屏蔽了底层硬件的复杂性和操作细节。

TK8710 HAL内部集成了TRM（Transceiver Radio Management）无线资源管理层和PHY Driver物理驱动层，提供芯片初始化、时隙配置、数据收发、状态监控等核心硬件管理功能。通过标准化的API接口，开发者可以轻松实现这些功能，而无需直接操作底层寄存器或处理复杂的硬件时序。

### 2.1 软件架构

TK8710驱动的软件架构采用分层设计，自上而下包含MAC层、TK8710 HAL层和TK8710硬件层三个主要组成部分。

![image-20260312150625598](D:/Windsurf_workspace/RK3506/tk8710_hal_v1.3/docs/assets/image-20260312150625598.png)

**MAC层（Media Access Control Layer）**：媒体接入控制层，负责网络协议栈的高层功能，包括数据包的封装与解封装、地址管理、流量控制、网络拓扑维护等。MAC层通过调用HAL层提供的标准接口来实现与TK8710芯片的通信，无需关心底层硬件的具体实现细节。

**TK8710 HAL**：硬件抽象层，位于MAC层和硬件之间的中间件层。HAL层为上层应用提供统一、简洁的编程接口，屏蔽了底层硬件的复杂性。内部集成了TRM无线资源管理层和PHY Driver物理驱动层，提供芯片初始化、时隙配置、数据收发、状态监控等核心硬件管理功能。

**TK8710芯片**：多天线数字收发器芯片，支持8组射频通道和128个用户的并发处理。芯片集成了先进的数字信号处理单元、多天线信号处理和波束成形技术，通过SPI接口与主控芯片通信，支持400MHz~960MHz频段范围内的无线通信。

### 2.2 HAL层功能说明

HAL内部集成了TRM无线资源管理层和PHY Driver物理驱动层，主要提供以下核心功能：

| 功能模块                | 说明                                                      | 负责模块        |
| ----------------------- | --------------------------------------------------------- | --------------- |
| **初始化管理**    | 芯片初始化、射频配置、日志系统配置、TRM系统初始化         | PHY Driver、TRM |
| **时隙配置/管理** | TDD帧结构配置、速率模式设置、天线配置、超帧配置/管理      | TRM             |
| **数据收发**      | 广播数据发送、用户数据接收/发送、发送队列管理、缓冲区管理 | TRM、PHY Driver |
| **状态监控**      | 运行状态查询、统计信息获取、信号质量监测                  | TRM、PHY Driver |
| **系统控制**      | 芯片启动、复位、调试（寄存器读写、校准等）                | TRM、PHY Driver |
| **波束管理**      | 用户波束信息获取、波束资源配置、天线波束调度              | TRM             |

### 2.4 HAL工作机制

TK8710 HAL采用基于TDD（时分双工）的通信机制，通过时隙管理和超帧调度实现高效的多用户并发通信。

#### 2.4.1 时隙结构

![image-20260313161147096](D:/Windsurf_workspace/RK3506/tk8710_hal_v1.3/docs/assets/image-20260313161147096.png)

基础帧结构由以下几类时隙构成：

 **BCN 时隙：**网关发送信标帧，用于终端获取网络时间与频率同步信息；

 **下行时隙：**网关向所有终端下发公共控制与广播数据、或向特定终端下发单播业务数据或控制指 令；

**上行时隙：**终端按调度结果或接入策略向网关发送业务数据；

#### 2.4.2 超帧机制

![image-20260313161250237](D:/Windsurf_workspace/RK3506/tk8710_hal_v1.3/docs/assets/image-20260313161250237.png)

超帧用于在更长时间尺度上组织 TDD 帧：

超帧用于在更长时间尺度上，组织多个TDD帧，支持多样化应用（扩容或多速率）：

**单速率超帧：**将相同速率、相同或不同帧周期的TDD帧组合在一起，为不同终端组或业务类型分配 不同帧资源；

**多速率超帧：**将不同速率和帧周期的TDD帧组合在一起，在同一超帧中支持多种速率的终端或业务，提升系统灵活性。

通过统一的绝对时间基准保持超帧级同步，从而实现：

- 保证终端在网关间漫游时的切换平滑；
- 降低同频干扰与碰撞概率；
- 为集中式时隙与频谱调度提供时间基准。

关于超帧的更加详细内容，请参看TurMass™ MAC协议的相关文档。

#### 2.4.3 数据收发机制

HAL层的数据收发采用异步回调机制：

- **数据发送**：MAC层调用发送接口将数据加入发送队列，HAL层在指定时隙序号通过SPI接口将数据发送给TK8710芯片
- **数据接收**：TK8710芯片接收到数据后通过中断通知HAL层，HAL层解析数据并通过回调函数通知MAC层
- **缓冲管理**：HAL层内部维护发送和接收缓冲区，实现数据的缓存和调度

#### 2.4.4 波束管理机制

HAL层采用基于哈希表的波束管理机制，为每个用户维护独立的波束配置信息：

- **波束信息存储**：TRM层使用4096大小的哈希表存储用户波束信息，支持最多128个并发用户。每个波束信息包含用户ID、频率、AH数据（8天线×2(I/Q)）、Pilot功率等参数。
- **波束分配策略**：支持广播波束和指定波束两种类型。广播波束用于系统全网覆盖，指定波束用于特定用户的数据传输，实现多天线波束成形和信号优化。
- **动态管理**：波束信息支持动态更新和超时清理。系统会自动检查波束信息的时效性，过期波束会被标记为无效，确保波束资源的有效利用。
- **查询接口**：提供波束信息查询接口，MAC层可以根据用户ID快速获取对应的波束配置，用于数据发送时的波束选择和参数设置。

#### 2.4.5 广播发送机制

HAL层支持两种广播发送模式：

- **无Payload广播**：HAL中TRM层基于MAC层配置的网络参数，自行管理超帧周期，在每个超帧中自动更新当前超帧序号并循环发送系统信息
- **带Payload广播**：HAL中TRM层在自行管理超帧基础上，根据MAC层指令在当前广播中插入Payload数据发送一次

## 3. TK8710 HAL API说明

### 3.1 TK8710 PHY API详细说明

**TK8710 PHY API汇总**

| 接口                  | 功能描述                           |
| --------------------- | ---------------------------------- |
| TK8710PhyInit         | 初始化TK8710芯片，配置基本工作参数 |
| TK8710PhyConfig       | 配置芯片工作时隙和通信参数         |
| TK8710PhyStart        | 启动芯片                           |
| TK8710PhyReset        | 复位芯片                           |
| TK8710PhySendData     | 发送数据                           |
| TK8710PhyReceiveData  | 接收数据                           |
| TK8710PhySetBeam      | 设置发送用户波束信息               |
| TK8710PhyGetBeam      | 获取接收用户波束信息               |
| TK8710PhyGetRxQuality | 获取接收信号质量信息               |
| TK8710PhyDebug        | 调试控制（发送Tone、校准等）       |
| TK8710PhyReadReg      | 读寄存器                           |
| TK8710PhyWriteReg     | 写寄存器                           |
| TK8710PhySetCbs       | 注册中断回调函数                   |

| callbacks | const TK8710DriverCbs* | 回调函数结构体指针 |

**回调函数结构体定义**:

```c
/* 专用回调函数类型 */
typedef void (*TK8710RxDataCb)(TK8710IrqResult* irqResult);
typedef void (*TK8710TxSlotCb)(TK8710IrqResult* irqResult);
typedef void (*TK8710SlotEndCb)(TK8710IrqResult* irqResult);
typedef void (*TK8710ErrorCb)(TK8710IrqResult* irqResult);

/* Driver回调结构体 */
typedef struct {
    TK8710RxDataCb onRxData;      /* 接收数据回调 */
    TK8710TxSlotCb onTxSlot;      /* 发送时隙回调 */
    TK8710SlotEndCb onSlotEnd;    /* 时隙结束回调 */
    TK8710ErrorCb onError;        /* 错误处理回调 */
} TK8710DriverCbs;
```

**支持的回调类型**:

| 回调函数  | 触发条件                 | 参数说明                    | 说明             |
| --------- | ------------------------ | --------------------------- | ---------------- |
| onRxData  | 接收到数据时触发         | irqResult: 包含接收数据信息 | 处理接收到的数据 |
| onTxSlot  | 发送时隙完成时触发       | irqResult: 包含发送结果信息 | 处理发送完成事件 |
| onSlotEnd | 时隙结束时触发           | irqResult: 包含时隙状态信息 | 处理时隙切换事件 |
| onError   | 硬件错误或通信失败时触发 | irqResult: 包含错误信息     | 处理错误恢复     |

**使用示例**:

```c
/* 定义接收数据回调 */
void OnRxData(TK8710IrqResult* irqResult)
{
    printf("接收到数据，用户索引: %u, 长度: %u\n", 
           irqResult->userIndex, irqResult->dataLen);
  
    if (irqResult->data && irqResult->dataLen > 0) {
        printf("数据内容: ");
        for (uint16_t i = 0; i < irqResult->dataLen && i < 16; i++) {
            printf("%02X ", irqResult->data[i]);
        }
        if (irqResult->dataLen > 16) printf("...");
        printf("\n");
    }
}

/* 定义发送时隙回调 */
void OnTxSlot(TK8710IrqResult* irqResult)
{
    printf("发送时隙完成，用户索引: %u, 状态: %u\n", 
           irqResult->userIndex, irqResult->status);
}

/* 定义时隙结束回调 */
void OnSlotEnd(TK8710IrqResult* irqResult)
{
    printf("时隙结束，时隙类型: %u\n", irqResult->slotType);
}

/* 定义错误处理回调 */
void OnError(TK8710IrqResult* irqResult)
{
    printf("硬件错误，状态: %u, 用户索引: %u\n", 
           irqResult->status, irqResult->userIndex);
    /* 执行错误恢复操作 */
}

/* 注册回调函数 */
TK8710DriverCbs callbacks = {
    .onRxData = OnRxData,
    .onTxSlot = OnTxSlot,
    .onSlotEnd = OnSlotEnd,
    .onError = OnError
};

TK8710RegisterCallbacks(&callbacks);
```

**说明**:

- 回调函数在中断上下文中执行，应保持简短高效
- 支持注册多个回调函数，后注册的会覆盖先注册的
- 所有回调函数都通过 `TK8710IrqResult`结构体传递事件信息
- 主要用于需要直接处理硬件事件的场景

#### 3.2.4 TK8710HalSendData - 发送数据

```c

    35,                       /* 发送功率 */

    0,                        /* 速率模式（广播忽略） */

}
```

### 5.3 常见问题FAQ

#### Q1: TK8710HalInit返回ERROR_INIT，如何排查？

**A**: 按以下步骤排查：

1. 检查SPI硬件连接是否正确
2. 确认SPI时钟频率不超过16MHz
3. 检查芯片供电是否正常
4. 使用示波器检查SPI信号波形

#### Q2: 发送数据后没有收到回调通知？

**A**: 可能原因：

1. 回调函数未正确注册，检查配置结构体中的callbacks配置
2. 目标设备不在线或无法接收
3. 发送功率过低，尝试增大txPower参数

#### Q3: 如何提高数据吞吐量？

**A**: 建议：

1. 使用更高的速率模式（如RATE_MODE_11）
2. 减小帧周期（frameTimeLen）
3. 批量发送数据，减少API调用次数
4. 确保发送队列不会满

#### Q4: 如何处理芯片异常？

**A**: 推荐流程：

```c
/* 1. 获取状态检查 */
TRM_Stats stats;
TK8710HalGetStatus(&stats);
if (stats.state == TRM_STATE_ERROR) {
    /* 2. 复位HAL */
    TK8710HalReset();
  
    /* 3. 重新初始化 */
    TK8710HalInit(NULL);
    TK8710HalConfig(NULL);
    TK8710HalStart();
}
```
